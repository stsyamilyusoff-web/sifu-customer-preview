Onboarding + Auth · all six screens, one skin, lighter front door
Cluster A of the secondary sweep. The six onboarding and auth screens, each before and after, brought onto the parent design language that already lives in the app's code, plus the cheap fixes and a lighter first wall. For Hafiz · 26 June 2026.
The parent skin already exists in code (the design tokens + the redesigned Invoice / Requests / Profile screens). These six screens are still on the old look. This brings them onto one language, ports the OTP fix the tutor app already shipped, restores the intro copy (commented out today), and splits the heaviest gate in the app: UpdateProfile asked for a full profile (about 13 taps) before the parent saw any value.
